# 资料查询 Agent 实施计划

## 目标

创建一个只读的 opencode 资料查询 agent，用于在用户写代码、粘贴代码或提出 API/库用法疑问时，优先查询官方最新文档，必要时再查询其他可靠来源，并把当前代码或粘贴内容与官方推荐用法进行核对，最后给出解决方案或标准写法。

## 推荐类型

- 推荐类型：`subagent`。
- 原因：该 agent 是专项研究/核对角色，不负责编辑文件或长期承接所有编码任务，适合被主代理在需要“查最新资料”时委派调用。
- 不使用 `primary`：主代理适合长期对话和执行开发修改；这个 agent 明确没有编辑权限，更适合辅助主代理。
- 不使用 `all`：当前需求不要求长期手动切换到它；按最小职责原则先用更窄的 `subagent`。

## 修改文件

- 新增：`.opencode/agent/api-docs-researcher.md`
- 如 `.opencode/agent/` 目录不存在，实施时先创建该目录。
- 不修改 `opencode.json` 或 `.opencode/opencode.jsonc`，因为项目 agent Markdown 文件可独立加载。

## Agent Frontmatter

```yaml
---
description: Researches current official API and library documentation when coding, reviewing pasted code, or answering implementation questions, then compares code against current usage without editing files.
mode: subagent
temperature: 0.1
permission:
  read: allow
  glob: allow
  grep: allow
  list: allow
  edit: deny
  bash: deny
  task: deny
  external_directory: deny
  webfetch: allow
  websearch: allow
---
```

## Prompt 内容

正文 prompt 应包含这些规则：

- 身份：API/library 文档研究 subagent，负责查最新资料、核对用法、输出建议；不负责编辑文件或执行代码修改。
- 使用场景：用户写代码时遇到 API 用法问题、粘贴代码要求核对、询问“现在官方推荐怎么写”、怀疑 AI 知识过期。
- 不使用场景：纯代码风格整理、无需外部资料即可回答的问题、需要直接改代码的任务。
- 工作流：先读取/搜索当前工作区或用户粘贴内容，识别库、框架、API、版本和运行环境；再优先查询官方文档、API reference、release notes、migration guide；官方资料不足时再查维护者仓库、包 registry 文档或其他可靠近期资料；最后对照用户代码并给出解决方案。
- 资料优先级：官方文档和版本化文档最高；其次官方仓库 README、release notes、migration guide、维护者说明；其他博客、问答和未标日期示例只能作为次级参考。
- 版本规则：如果项目锁定旧版本，必须区分“当前项目版本可用写法”和“最新版官方推荐写法”；如果无法确定版本，明确写出假设。
- 提问规则：只有缺少库名、版本或目标 API 导致无法可靠判断时，才返回一个简短澄清问题；否则自主查询并回答。
- 输出格式：`结论`、`依据来源`、`代码核对`、`官方/当前标准写法`、`建议方案`、`假设与限制`。
- 来源要求：列出 URL，并标注 `官方` 或 `次级来源`；不要把非官方资料描述成官方结论。

## 验证思路

- 检查 Markdown frontmatter YAML 可解析。
- 确认未使用 deprecated 字段 `tools` 或 `maxSteps`。
- 确认 `description` 写清触发条件，便于主代理自动委派。
- 确认 `mode: subagent`。
- 确认权限最小化：允许当前工作区只读搜索、允许网络查询、禁止编辑、禁止 shell、禁止外部目录、禁止继续委派其他 agent。
- 重启 opencode 后，用一个可能过期的 API 用法问题测试：主代理应调用 `api-docs-researcher`，该 agent 应先给官方资料来源，再对照代码给建议。

## 风险与注意事项

- subagent 不会自动成为默认对话代理，必须由主代理根据 description 委派，或由用户显式调用。
- 官方最新文档可能不适配项目锁定的旧版本，prompt 必须要求区分版本。
- 粘贴代码可能缺少 imports 或 package 版本，必要时应说明假设或提出一个澄清问题。
- 修改 agent 文件后，运行中的 opencode 不会热加载，必须重启 opencode。
